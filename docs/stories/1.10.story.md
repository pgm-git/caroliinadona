## Descricao

Configurar pipeline de CI/CD com GitHub Actions para garantir qualidade de código em cada push e pull request. Inclui lint (ESLint), type checking (TypeScript), build de produção e futuramente testes automatizados.

## Criterios de Aceite

- [ ] Pipeline roda em: push para `main` e em todo pull request
- [ ] Step 1: Install dependencies (`npm ci`)
- [ ] Step 2: Lint (`npm run lint`) — ESLint com regras do Next.js
- [ ] Step 3: Type check (`npm run typecheck`) — `tsc --noEmit`
- [ ] Step 4: Build (`npm run build`) — build de produção
- [ ] Step 5: Test placeholder (`npm test`) — script que passa (sem testes ainda)

## Notas de Implementacao

- GitHub Actions workflow:
  ```yaml
  on:
    push:
      branches: [main]
    pull_request:
      branches: [main]
  jobs:
    ci:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v4
        - uses: actions/setup-node@v4
          with:
            node-version: 20
            cache: 'npm'
        - run: npm ci
        - run: npm run lint
        - run: npm run typecheck
        - run: npm run build
        - run: npm test
  ```
- ESLint rules:
  - Extends: `next/core-web-vitals`, `next/typescript`
  - Rules adicionais: `no-unused-vars` (warn), `@typescript-eslint/no-explicit-any` (warn)
- TypeScript: `strict: true` já configurado em 1.1
- Cache do GitHub Actions economiza ~30-60s no install
- Husky é opcional no MVP — pode ser adicionado depois se o time preferir
- O step de test é placeholder: `echo "No tests yet" && exit 0`
