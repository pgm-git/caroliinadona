## Notas de Implementacao

- Upstash Redis requer TLS — usar `ioredis` com `tls: {}` e `maxRetriesPerRequest: null` para BullMQ
- Configuração de retry por fila:
  - `document-processing`: 3 attempts, backoff exponencial 5s
  - `petition-generation`: 2 attempts, backoff exponencial 10s
  - `batch-import`: 1 attempt, prioridade 10 (baixa)
  - `index-sync`: cron `0 6 * * *` (diário às 6am)
- Workers devem rodar em processo separado em produção (Railway), mas em dev podem rodar no mesmo processo
- BullBoard deve ser protegido — apenas admin pode acessar `/api/queues`
- Dead Letter Queue (DLQ): jobs que falharam após max retries vão para fila `{name}-dlq`
- Connection pooling: reutilizar conexão Redis entre filas
