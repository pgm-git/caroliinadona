## Descricao

Permitir que o usuario edite campos extraidos diretamente na tela de revisao (inline editing). Campos editados manualmente devem ter a confianca ajustada para 100% e serem marcados como "corrigido manualmente". O usuario tambem pode reprocessar um documento com parametros ajustados.

## Criterios de Aceite

- [ ] Campos extraidos editaveis via click/double-click
- [ ] Ao editar, campo muda para input editavel com botoes Salvar/Cancelar
- [ ] Campo editado manualmente marcado com badge "Corrigido" (azul)
- [ ] Confianca do campo editado ajustada para 100%
- [ ] tRPC mutation para salvar edicoes parciais (campo individual)
- [ ] tRPC mutation para reprocessar documento (limpar + re-enfileirar)
- [ ] Historico de versoes: nova extracao nao sobrescreve anterior (cria novo registro)
- [ ] Botao "Reprocessar" com confirmacao modal
- [ ] Toast de sucesso/erro nas operacoes

## Tarefas Tecnicas

- [ ] Criar componente `EditableField` em `src/components/extraction/editable-field.tsx`
- [ ] Adicionar tRPC mutation `extraction.updateField` para edicao individual

## Notas de Implementacao

### Edicao de Campo

```typescript
// mutation input
z.object({
  extractedDataId: z.string().uuid(),
  fieldPath: z.string(), // ex: "valorPrincipal", "devedor.nome"
  newValue: z.union([z.string(), z.number(), z.null()]),
})

// No DB: atualizar extractedFields JSONB parcialmente
await db.update(extractedData).set({
  extractedFields: sql`jsonb_set(
    ${extractedData.extractedFields},
    ${fieldPath}::text[],
    ${JSON.stringify({ value: newValue, confidence: 100, manuallyEdited: true })}::jsonb
  )`,
  updatedAt: new Date(),
}).where(eq(extractedData.id, extractedDataId));
```
