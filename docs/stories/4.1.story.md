## Descricao

Implementar o motor de validação com regras para verificar completude de dados extraídos e validação algorítmica de CPF/CNPJ. Cada regra tem severidade (BLOCKING, WARNING, INFO) e gera registros na tabela `validations`.

## Criterios de Aceite

- [ ] Service `ValidationService` com engine de regras extensível
- [ ] Validação de CPF com dígitos verificadores
- [ ] Validação de CNPJ com dígitos verificadores
- [ ] Regras de completude: devedor CPF/CNPJ obrigatório, valor principal obrigatório, datas obrigatórias
- [ ] Cada regra com severidade: BLOCKING (red), WARNING (yellow), INFO (green)
- [ ] Resultados persistidos na tabela `validations`
- [ ] Método `validateCase(caseId)` que executa todas as regras
- [ ] tRPC router `validation.ts` com procedures: runValidation, listByCaseId, resolveValidation

## Tarefas Tecnicas

- [ ] Criar `src/lib/validation/cpf-cnpj.ts` com validadores puros
- [ ] Criar `src/lib/validation/rules.ts` com definição de regras
- [ ] Criar `src/server/services/validation.service.ts`
- [ ] Criar `src/server/trpc/routers/validation.ts`
- [ ] Adicionar validation router ao root.ts
