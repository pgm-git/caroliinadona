## Descricao

Implementar o worker BullMQ que orquestra o pipeline completo de processamento: download do storage → OCR → extracao → classificacao → persistencia. O worker ja existe como stub em `src/server/queue/workers/document.worker.ts` e precisa ser conectado aos servicos reais de OCR e extracao.

## Criterios de Aceite

- [ ] Worker processa jobs com actions: `ocr`, `classify`, `extract`, `full` (pipeline completo)
- [ ] Action `full` executa: OCR → classificacao → extracao → identificacao de partes
- [ ] `processingStartedAt` setado ao iniciar, `processingCompletedAt` ao finalizar
- [ ] Em caso de erro: `processingError` preenchido com mensagem descritiva
- [ ] Retry com backoff exponencial (3 tentativas, delay 5s)
- [ ] Concurrency de 2 workers em paralelo
- [ ] Logs estruturados de cada etapa do pipeline
- [ ] Job progress atualizado (0%, 25%, 50%, 75%, 100%)

## Tarefas Tecnicas

- [ ] Atualizar `src/server/queue/workers/document.worker.ts` com processamento real
- [ ] Implementar handler para action `ocr` (chama OcrService)
- [ ] Implementar handler para action `classify` (chama ExtractionService.classify)
- [ ] Implementar handler para action `extract` (chama ExtractionService.extract)

## Notas de Implementacao

### Pipeline Completo (action: "full")

```typescript
async function processFullPipeline(job: Job<DocumentProcessingJob>) {
  const { documentId, caseId, orgId } = job.data;

  // 1. Marcar inicio
  await db.update(documents).set({
    processingStartedAt: new Date(),
    processingError: null,
  }).where(eq(documents.id, documentId));

  await job.updateProgress(0);

  // 2. OCR
  const ocrResult = await ocrService.processOcr(documentId, orgId);
  await job.updateProgress(25);

  // 3. Classificacao
  const classification = await extractionService.classifyDocumentType(ocrResult.text);
  await db.update(documents).set({
    aiClassification: classification.type,
    aiConfidence: classification.confidence.toString(),
  }).where(eq(documents.id, documentId));
  await job.updateProgress(50);

  // 4. Extracao estruturada
  await extractionService.extractAndPersist(documentId, caseId, orgId, ocrResult.text, classification.type);
  await job.updateProgress(75);

  // 5. Identificacao de partes (Story 3.4)
  // await partyService.identifyParties(...)

  // 6. Marcar como processado
  await db.update(documents).set({
    isProcessed: true,
    processingCompletedAt: new Date(),
  }).where(eq(documents.id, documentId));
  await job.updateProgress(100);
}
```
