## Descricao

Implementar o servico de identificacao e classificacao de partes processuais a partir dos dados extraidos. O servico deve identificar: devedor principal, avalistas/fiadores, conjuges (quando aplicavel), credor (cooperativa). Cada parte deve ser associada ao seu CPF/CNPJ e endereco, e persistida na tabela `parties`.

## Criterios de Aceite

- [ ] Service `PartyService` em `src/server/services/party.service.ts`
- [ ] Identificacao de devedor, avalista, fiador, credor a partir dos dados extraidos
- [ ] Classificacao do `role` conforme enum: CREDOR, DEVEDOR, AVALISTA, FIADOR, CODEVEDORA, REPRESENTANTE_LEGAL
- [ ] `personType` classificado como PF (pessoa fisica) ou PJ (pessoa juridica) baseado em CPF vs CNPJ
- [ ] Devedor principal marcado com `isPrimary = true`
- [ ] Dados persistidos na tabela `parties` com todos os campos disponiveis
- [ ] Deduplicacao: se parte com mesmo CPF/CNPJ ja existe no caso, atualizar ao inves de criar
- [ ] Integrado ao pipeline do worker (step apos extracao)

## Tarefas Tecnicas

- [ ] Criar `src/server/services/party.service.ts`
- [ ] Implementar `identifyParties(extractedFields, caseId, orgId)`
- [ ] Implementar mapeamento de campos extraidos para schema da tabela `parties`
- [ ] Implementar deduplicacao por CPF/CNPJ dentro do caso

function mapExtractedToParty(extracted: CcbExtraction, role: string) {
  if (role === 'devedor') {
    return {
      role: 'DEVEDOR',
      isPrimary: true,
      fullName: extracted.devedor.nome.value,
      cpf: extracted.devedor.cpfCnpj.value,
      personType: isCnpj(extracted.devedor.cpfCnpj.value) ? 'PJ' : 'PF',
      // ... endereco, etc
    };
  }
  // similar para avalista, credor
}
