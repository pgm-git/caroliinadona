## Descricao

Implementar o fluxo completo de gerenciamento de casos: criacao de caso a partir de documentos enviados, listagem paginada com filtros, e tela de detalhe do caso. O caso e a entidade central do Carolina — todo o fluxo de processamento gira em torno dele. A criacao de caso deve ser integrada com o upload de documentos (2.1 + 2.2), criando automaticamente o caso e associando os documentos enviados.

### Fluxo de Criacao

1. Validar input do formulario
2. Gerar referencia interna
3. Inserir caso na tabela `cases` (status: `received`)
4. Para cada documento: fazer upload via StorageService (2.2) e inserir na tabela `documents`
5. Para cada documento: enfileirar job no BullMQ (`document-processing` queue)
6. Registrar no `status_history` (null → `received`)
7. Retornar caso criado com documentos

### Tela de Detalhe

A pagina de detalhe exibe:
- **Header:** Titulo, referencia interna, status (Badge colorido), tipo, prioridade
- **Secao Documentos:** Lista de documentos com nome, tipo, tamanho, status de processamento
- **Secao Timeline:** Historico de status do caso (status_history)
- **Secao Dados:** Dados financeiros quando disponíveis (preenchidos em EPICs posteriores)
- **Acoes:** Botao de adicionar documentos, editar caso

### Integracao com BullMQ

Ao criar o caso, para cada documento uploaded:

```typescript
import { getQueue, QUEUE_NAMES } from "@/server/queue";

const queue = getQueue(QUEUE_NAMES.DOCUMENT_PROCESSING);
await queue.add("process-document", {
  documentId: doc.id,
  caseId: case.id,
  orgId: orgId,
  action: "ocr" as const,
}, {
  priority: case.priority,
});
```
