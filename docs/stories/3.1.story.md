## Descricao

Implementar o client do Google Document AI para processamento OCR de documentos. O client deve receber um documento (PDF ou imagem) do Supabase Storage e retornar o texto extraido com metadados de confianca. Deve suportar documentos impressos (acuracia >= 97%) e manuscritos (acuracia >= 85%).

## Criterios de Aceite

- [ ] Client Google Document AI configurado em `src/server/services/ocr.service.ts`
- [ ] Funcao `processOcr(documentId)` que baixa do storage e envia para Document AI
- [ ] Retorno tipado com texto extraido, confianca geral e confianca por pagina
- [ ] Texto OCR salvo no campo `ocrText` da tabela `documents`
- [ ] Confianca salva no campo `aiConfidence` da tabela `documents`
- [ ] Suporte a PDF e imagens (JPEG, PNG, TIFF)
- [ ] Variavel de ambiente `GOOGLE_APPLICATION_CREDENTIALS` ou `GOOGLE_DOCUMENT_AI_PROCESSOR_ID`
- [ ] Tratamento de erros com mensagem descritiva em `processingError`
- [ ] Logs estruturados do processamento

## Notas de Implementacao

### Google Document AI Setup

```typescript
import { DocumentProcessorServiceClient } from '@google-cloud/documentai';

const client = new DocumentProcessorServiceClient();

// Processor: projects/{project}/locations/{location}/processors/{processor}
const processorName = process.env.GOOGLE_DOCUMENT_AI_PROCESSOR_ID;

async function processDocument(content: Buffer, mimeType: string) {
  const [result] = await client.processDocument({
    name: processorName,
    rawDocument: { content: content.toString('base64'), mimeType },
  });
  return {
    text: result.document?.text ?? '',
    confidence: result.document?.pages?.reduce((sum, p) => sum + (p.confidence ?? 0), 0)
      / (result.document?.pages?.length || 1),
    pages: result.document?.pages?.map(p => ({
      pageNumber: p.pageNumber,
      confidence: p.confidence,
    })),
  };
}
```
