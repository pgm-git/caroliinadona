# Story 1.4: Implementar autenticação (login, registro, logout)

**Epic:** EPIC-01 Foundation
**Prioridade:** P0
**Estimativa:** 5 pontos

## Descricao

Implementar fluxo completo de autenticação usando Supabase Auth: registro de novo escritório (organização + usuário admin), login com email/senha, logout, recuperação de senha e proteção de rotas. Interface 100% em PT-BR (NFR-017).

## Criterios de Aceite

- [ ] Página de login (`/login`) com email e senha, validação de campos, feedback de erro
- [ ] Página de registro (`/register`) criando organização + usuário admin automaticamente
- [ ] Logout funcional com limpeza de sessão e redirect para `/login`
- [ ] Página de recuperação de senha (`/forgot-password`) com envio de email
- [ ] Página de redefinição de senha (`/reset-password`) com token de URL
- [ ] Middleware Next.js protegendo rotas `/dashboard/*` — redireciona para `/login` se não autenticado
- [ ] Sessão persistente (refresh token automático via middleware Supabase)
- [ ] Interface 100% em PT-BR (labels, mensagens de erro, placeholders)
- [ ] Registro cria `organization` + `users` + `org_members` no banco
- [ ] Loading states em todos os botões de submit
- [ ] Tratamento de erros: credenciais inválidas, email já cadastrado, email não confirmado

## Notas de Implementacao

- Usar `@supabase/ssr` para gerenciamento de sessão server-side
- No registro, criar organização e membro dentro de uma transação (usar Supabase RPC ou service role client)
- O `user.id` do Supabase Auth deve ser o mesmo `users.id` no banco (UUID compartilhado)
- Usar `useTransition` do React para loading states nos forms
- Validação com Zod nos formulários
- Mensagens de erro em PT-BR:
  - "Email ou senha inválidos"
  - "Este email já está cadastrado"
  - "Confirme seu email antes de fazer login"
  - "Preencha todos os campos obrigatórios"
