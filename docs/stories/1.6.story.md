# Story 1.6: Configurar tRPC (router raiz, context, middleware auth)

**Epic:** EPIC-01 Foundation
**Prioridade:** P0
**Estimativa:** 3 pontos
**Status:** Ready
**Depende de:** 1.1, 1.4, 1.5
**Bloqueia:** EPIC-02 a EPIC-07 (todos os módulos usam tRPC)

## Descricao

Configurar tRPC v11 para comunicação type-safe entre frontend e backend. Criar router raiz, context com autenticação/org_id, middleware de auth e RBAC, provider no client e endpoint handler no API route. Preparar a estrutura para que cada módulo de negócio crie seu próprio sub-router.

## Criterios de Aceite

- [ ] tRPC v11 instalado e configurado (`@trpc/server`, `@trpc/client`, `@trpc/react-query`, `@trpc/next`)
- [ ] Router raiz (`appRouter`) criado com type export
- [ ] Context inclui: `user`, `orgId`, `role`, `supabase` (server client)
- [ ] Procedure base: `publicProcedure` (sem auth) e `protectedProcedure` (com auth obrigatória)
- [ ] Middleware de auth que extrai user da sessão Supabase e injeta no context
- [ ] Middleware de RBAC composável: `.use(requireRole('admin'))`, `.use(requirePermission('cases', 'create'))`
- [ ] API handler em `/api/trpc/[trpc]/route.ts` (App Router handler)
- [ ] tRPC Provider configurado no client com TanStack Query
- [ ] Health check procedure funcional: `appRouter.health.ping` retorna `{ status: 'ok' }`
- [ ] Type inference funcionando end-to-end (hover no client mostra tipos do server)
- [ ] Error handling com códigos tRPC padrão (UNAUTHORIZED, FORBIDDEN, NOT_FOUND, etc.)

## Notas de Implementacao

- Usar tRPC v11 com `initTRPC.create()` pattern
- O context deve extrair org_id do user logado para todas as queries filtrarem por tenant automaticamente
- Usar `superjson` como transformer para suporte a Date, BigInt, etc.
- O error formatter deve mapear erros Zod para mensagens amigáveis em PT-BR
- Cada módulo futuro (intake, petition, etc.) criará seu próprio router em `src/modules/{name}/{name}.router.ts` e será mergeado no root
- Para Server Components, usar `createCallerFactory` para chamadas diretas sem HTTP
- O httpBatchLink deve apontar para `/api/trpc`
