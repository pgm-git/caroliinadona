## Notas de Implementacao

### Estrutura de diretorios

O projeto segue a arquitetura modular com separacao clara de responsabilidades:

```
packages/app/src/
├── app/                    # Next.js App Router pages
│   ├── (auth)/             # Grupo de rotas de autenticacao (login, register, forgot-password)
│   ├── (dashboard)/        # Grupo de rotas autenticadas (dashboard, cases, documents, etc.)
│   ├── layout.tsx          # Root layout com providers
│   └── page.tsx            # Landing page / redirect
├── modules/                # Feature modules (cada modulo com seus components, hooks, utils)
├── server/                 # Server-side: tRPC routers, services, repositories
├── components/
│   ├── ui/                 # shadcn/ui components (gerenciados pelo CLI)
│   └── layout/             # Layout components (Sidebar, Header, etc.)
└── lib/                    # Shared utilities, configs, types
```

### Design System - Cores

As cores do tema Carolina seguem a paleta definida na arquitetura:

- **Navy (#1E3A5F):** Cor primaria, usada em headers, sidebar, elementos de destaque principal
- **Blue (#2563EB):** Cor de acao, usada em botoes primarios, links, elementos interativos
- **Emerald (#059669):** Cor de sucesso, usada em status positivos, confirmacoes
- **Amber (#D97706):** Cor de aviso, usada em alertas, pendencias
- **Red (#DC2626):** Cor de erro/perigo, usada em erros, acoes destrutivas

### Workspace Monorepo

O `package.json` na raiz deve configurar workspaces para permitir a adicao futura de outros packages (como um package de tipos compartilhados ou um package de workers BullMQ):

```json
{
  "name": "carolina",
  "private": true,
  "workspaces": ["packages/*"]
}
```

### shadcn/ui

Os 9 componentes base cobrem as necessidades imediatas das stories seguintes. Componentes adicionais podem ser adicionados conforme necessario via `npx shadcn@latest add [componente]`. O helper `cn()` em `src/lib/utils.ts` e essencial para o funcionamento do shadcn e combina `clsx` com `tailwind-merge`.

### Next.js 15

Usar App Router (nao Pages Router). Configurar `next.config.ts` com:
- `images.remotePatterns` para dominio do Supabase Storage (configurar depois na story 1.2)
- Nao habilitar `output: 'standalone'` ainda (sera feito na story de deploy)
