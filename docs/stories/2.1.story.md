## Descricao

Criar o componente de upload de documentos com suporte a drag-and-drop, selecao multipla de arquivos, validacao de formato/tamanho na entrada e barra de progresso individual por arquivo. Este componente sera reutilizado em diversas telas do sistema (criacao de caso, adicao de documentos a caso existente). O componente deve validar arquivos antes do envio e fornecer feedback visual claro em portugues.

## Criterios de Aceite

- [ ] Componente `FileUploader` com area de drag-and-drop visualmente clara
- [ ] Suporte a selecao multipla de arquivos (ate 50 por vez)
- [ ] Formatos aceitos: PDF, JPG, PNG, TIFF (rejeitar outros com mensagem clara)
- [ ] Tamanho maximo por arquivo: 25 MB (rejeitar com mensagem clara)
- [ ] Barra de progresso individual por arquivo durante upload
- [ ] Estados visuais: idle, dragover, uploading, success, error
- [ ] Feedback visual com semaforo: verde (valido), vermelho (rejeitado), azul (uploading)
- [ ] Botao para remover arquivo da lista antes de confirmar envio
- [ ] Listagem de arquivos selecionados com nome, tamanho e status
- [ ] Mensagens de erro em PT-BR (ex: "Arquivo muito grande. Maximo: 25 MB")
- [ ] Componente acessivel (WCAG 2.1 AA): navegavel por teclado, labels adequados
- [ ] Responsive: funciona em desktop (1280px+) e tablet (768px+)

## Notas de Implementacao

### Constantes de Upload

```typescript
export const UPLOAD_CONFIG = {
  maxFileSize: 25 * 1024 * 1024, // 25 MB
  maxFiles: 50,
  acceptedMimeTypes: [
    "application/pdf",
    "image/jpeg",
    "image/png",
    "image/tiff",
  ],
  acceptedExtensions: [".pdf", ".jpg", ".jpeg", ".png", ".tif", ".tiff"],
} as const;
```

### Validacao de Arquivo

```typescript
interface FileValidationResult {
  valid: boolean;
  error?: string;
}

function validateFile(file: File): FileValidationResult {
  if (file.size > UPLOAD_CONFIG.maxFileSize) {
    return { valid: false, error: `Arquivo muito grande. Maximo: 25 MB` };
  }
  if (!UPLOAD_CONFIG.acceptedMimeTypes.includes(file.type)) {
    return { valid: false, error: `Formato nao aceito. Use: PDF, JPG, PNG ou TIFF` };
  }
  return { valid: true };
}
```

### Hook useFileUpload

O hook gerencia o estado de todos os arquivos selecionados:

```typescript
interface UploadFile {
  id: string;           // UUID gerado no cliente
  file: File;
  status: "pending" | "uploading" | "success" | "error";
  progress: number;     // 0-100
  error?: string;
  validationResult: FileValidationResult;
}

interface UseFileUploadReturn {
  files: UploadFile[];
  addFiles: (files: FileList | File[]) => void;
  removeFile: (id: string) => void;
  clearFiles: () => void;
  validFiles: UploadFile[];
  invalidFiles: UploadFile[];
  isUploading: boolean;
  uploadFiles: (onUpload: (file: File, onProgress: (pct: number) => void) => Promise<string>) => Promise<void>;
}
```

### Componente FileUploader

```tsx
interface FileUploaderProps {
  onFilesReady?: (files: File[]) => void;
  onUpload?: (file: File, onProgress: (pct: number) => void) => Promise<string>;
  maxFiles?: number;

  className?: string;
}
```

- A area de drop usa `onDragEnter`, `onDragOver`, `onDragLeave`, `onDrop`
- Usar `<input type="file" multiple accept=".pdf,.jpg,.jpeg,.png,.tif,.tiff">` oculto
- Clicar na area de drop aciona o input file
- Ao arrastar sobre a area, mudar estilo para `border-blue-500 bg-blue-50`

### Acessibilidade

- Area de drop com `role="button"`, `tabIndex={0}`, `aria-label="Arraste arquivos ou clique para selecionar"`
- Input file oculto com label acessivel
- Status de cada arquivo anunciado com `aria-live="polite"`
- Botao de remover com `aria-label="Remover arquivo {filename}"`

### Design Visual

- Area de drop: borda tracejada (`border-dashed`), icone de upload centralizado
- Cores: Navy #1E3A5F para icone, Blue #2563EB para hover/dragover, Emerald #059669 para sucesso, Red #DC2626 para erro
- Lista de arquivos abaixo da area de drop com icone de tipo, nome, tamanho formatado
